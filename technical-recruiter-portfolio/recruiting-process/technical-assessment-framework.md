# Technical Assessment Framework

This framework is designed to help evaluate technical candidates in a structured and fair way.

## Assessment Areas

### 1. Technical Knowledge
Does the candidate demonstrate working knowledge of the required technical domain?

### 2. Troubleshooting Approach
Can the candidate break down problems logically and identify likely causes?

### 3. Communication
Can the candidate explain technical topics clearly to technical and non-technical audiences?

### 4. Depth vs Breadth
Does the candidate show hands-on depth in the most important skill areas?

### 5. Role Alignment
Does the candidate's background match the actual needs of the team?

## Example Screening Questions

### Cloud Support Engineer
- Tell me how you would troubleshoot an application connectivity issue.
- What is the difference between a security group and a network ACL?
- How would you respond to a production incident reported by a customer?

### Solutions Engineer
- How would you explain an API integration issue to a customer?
- What steps would you take to isolate the cause of a failed integration?
- How do you balance technical accuracy with customer communication?

## What I Look For

- Clear thinking
- Practical troubleshooting
- Strong communication
- Relevant technical depth
- Good judgment
