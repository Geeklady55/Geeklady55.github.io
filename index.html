What this project demonstrates

Recruiter-friendly, engineering-heavy: security, reliability, observability, AI grounding, and deployable packaging.

JWT Authentication

Protected endpoints + token expiry

Login endpoint issues JWTs; chat + upload endpoints require a valid token. Clear 403 responses for invalid/expired tokens.

RAG with Embeddings

Upload documents → retrieval → grounded context

Documents are chunked, embedded, indexed (FAISS), retrieved (top-k), and injected into the prompt for better accuracy.

Streaming Responses (SSE)

Token streaming for real-time UX

Server-Sent Events stream partial output so the UI feels responsive and production-ready.

Observability

Structured logs + request IDs

Every request is traceable with request IDs; upstream errors map to clean 502s; health endpoint supports monitoring.

Error Handling

Explicit, documented failure modes

422 validation errors, 403 auth failures, 502 upstream LLM failures, and 500 internal errors with safe responses.

Dockerized

Runs anywhere

Docker + docker-compose provide consistent execution across dev machines, demos, and deployment targets.

Why this stands out

It shows you can ship real systems: authentication, retrieval architecture, streaming UX, and operational discipline — not just prompt engineering.

Architecture

Clean separation of concerns: API routing, authentication, retrieval, and LLM provider abstraction.

High-level flow
Browser UI
  → (JWT) FastAPI /chat-stream
    → Persona router selects system prompt
    → RAG retrieves top-k chunks from FAISS
    → LLM call (OpenAI-compatible API)
    → Stream tokens back via SSE
Key endpoints
  • POST /login → returns JWT token
  • POST /upload → index documents (RAG)
  • POST /chat → non-stream reply
  • POST /chat-stream → streaming SSE reply
  • GET /health → service health

Design choices
  • Provider abstraction for easy LLM swaps
  • RAG layer isolated for future vector DB migration
  • Stateless API design (ready for scaling)
  • Explicit error mapping and safe responses

Run locally

This is written to be demo-friendly: one command to run, predictable endpoints, and clear steps.

Docker (recommended)
cp .env.example .env
# set OPENAI_API_KEY + JWT_SECRET

docker compose up --build

# Health check
curl http://localhost:8000/health
Example API usage
# 1) Login to get token
curl -X POST http://localhost:8000/login

# 2) Upload document (RAG)
curl -X POST http://localhost:8000/upload \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@docs/sample.txt"

# 3) Stream chat (SSE)
curl -N -X POST http://localhost:8000/chat-stream \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"persona":"integration_architect","messages":[{"role":"user","content":"Design an event-driven integration."}]}'
