3) Baseline (Before)

-- BEFORE (sample)
WITH base AS (
  SELECT o.customer_id, o.order_id, o.total, o.order_ts
  FROM orders o
  WHERE o.status IN ('PAID','SHIPPED')
    AND o.order_ts >= :start
    AND o.order_ts <  :end
)
SELECT customer_id,
       COUNT(*) AS orders,
       SUM(total) AS revenue
FROM base
GROUP BY customer_id
ORDER BY revenue DESC
LIMIT 50;

4) Diagnosis

Observed

Full table scans, heavy sort, or poor join order causing high IO / latency spikes.

Tools

EXPLAIN / EXPLAIN ANALYZE, query stats, and row estimates vs actual.

Hypothesis

Improve selectivity by indexing filter columns + reduce sort/aggregation cost.

5) Optimization (After)

Document what changed and why: indexes, query rewrite, predicate pushdown, reduced data movement. 

-- AFTER (sample improvements)
-- Index ideas (adapt to your DB engine):
-- CREATE INDEX idx_orders_status_ts ON orders(status, order_ts);

SELECT o.customer_id,
       COUNT(*) AS orders,
       SUM(o.total) AS revenue
FROM orders o
WHERE o.status IN ('PAID','SHIPPED')
  AND o.order_ts >= :start
  AND o.order_ts <  :end
GROUP BY o.customer_id
ORDER BY revenue DESC
LIMIT 50;
