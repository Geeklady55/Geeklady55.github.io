Linux Case Study: Systemd Service Debugging

This case study shows how I investigate a failed Linux service using systemctl, journald, permissions checks, and restart validation.

View Commands Back to Linux

Problem

A service repeatedly failed to start and entered a restart loop after a deployment.

Investigation

systemctl status myapp.service
journalctl -u myapp.service --since "1 hour ago"
systemctl show myapp.service -p ExecMainStatus -p NRestarts
ls -l /opt/myapp/bin/myapp

Fix

The ExecStart target did not have the correct execute permissions. After correcting permissions and reloading systemd, the service started successfully.

Validation

chmod +x /opt/myapp/bin/myapp
systemctl daemon-reload
systemctl restart myapp.service
systemctl status myapp.service
journalctl -u myapp.service --since "10 minutes ago"
